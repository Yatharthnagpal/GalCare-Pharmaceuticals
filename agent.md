# Galcare Website Development Guidelines

## Project Overview

Project Name: Galcare Website Redesign

Objective:
Transform the existing Galcare website into a modern, premium pharmaceutical website while preserving the company's existing brand identity.

Current Website:
https://www.galcare.com

Primary Design References:
- https://www.accredo.com/
- https://www.colgate.com/en-in
- https://zydususa.com/
- https://www.zs.com/

- https://www.lifecareneuro.com/

Additional UI Inspiration:
- https://pharmaceutical-company-website.vercel.app/

---

# Brand Identity

This is the MOST IMPORTANT requirement.

DO NOT redesign the company branding.

The existing Green + White color palette MUST remain the primary identity.

Allowed improvements:

- Better typography
- Better spacing
- Better gradients
- Better shadows
- Better iconography
- Better accessibility
- Better illustrations

Do NOT change the brand to blue.

Do NOT redesign the logo unless specifically requested.

---

# Company Divisions

Galcare has three business divisions.

Always keep them visible throughout the website.

1. Neuropsychiatric

2. Dermatology

3. Third Party Manufacturing

Each division should have its own dedicated landing page.

---

# Website Goals

The website should communicate:

- Trust
- Innovation
- Quality
- Scientific Excellence
- Modern Manufacturing
- Healthcare Leadership
- Global Presence

The design should feel comparable to multinational pharmaceutical companies while remaining uniquely Galcare.

---

# Existing Assets

Whenever possible:

- Reuse images from the current Galcare website.
- Preserve manufacturing photographs.
- Preserve laboratory photographs.
- Preserve product photographs.
- Preserve dermatology imagery.

Only replace assets if the quality is too poor.

# UI Principles

Follow:

- 8pt spacing
- Responsive design
- Large whitespace
- Rounded corners
- Soft shadows
- Premium typography
- Accessible contrast
- Smooth animations

---

# Animation Rules

Animations must be subtle.

Allowed:

- Fade
- Slide
- Reveal
- Counter
- Hover
- Image Zoom

Avoid excessive motion.

---

# Products Module

The products page must NOT be static.

Requirements:

- Database-driven
- Search
- Filters
- Categories
- Product Details
- Product Images
- Downloads
- Related Products
- Inquiry Form

Future-ready for thousands of products.

---

# SEO

Every page must include:

- Meta Title
- Meta Description
- OpenGraph
- Schema.org
- Breadcrumb
- Canonical URL

---

# Accessibility

Follow WCAG 2.2

- Keyboard navigation
- Proper heading hierarchy
- ARIA labels
- Screen reader compatibility

---

# Performance

Target Lighthouse Score

Performance >95

Accessibility >95

SEO >95

Best Practices >95

---

# Code Quality

Always:

- Write reusable components.
- Avoid duplicated code.
- Use TypeScript.
- Use modular architecture.
- Use clean folder structures.
- Use proper naming conventions.

Never hardcode repeated data.

---

# Development Workflow

Before writing code:

1. Understand the feature.
2. Check AGENT.md.
3. Check project PRD.
4. Reuse existing components.
5. Generate production-quality code.

---

# AI Behaviour

Never:

- Randomly change colors.
- Change branding.
- Remove existing business divisions.
- Replace authentic company content with generic placeholders.

Always:

- Preserve Galcare's identity.
- Improve UX.
- Keep Green + White branding.
- Follow enterprise software engineering practices.

---

# Success Criteria

The finished website should feel like a modern global pharmaceutical company's website while maintaining Galcare's original identity and values.
